# Product Owner — Priorytety Funkcji

## Priorytetyzacja MoSCoW

### MUST HAVE (zrealizowane)
| Funkcja | Priorytet biznesowy | Uzasadnienie |
|---|---|---|
| Katalog 15 gatunków | Krytyczny | Core produktu — bez tego nie ma herbarium |
| Modal szczegółów | Krytyczny | Edukacyjna wartość — szczegóły gatunku |
| Ecosystem Map | Wysoki | Unikalna funkcja wyróżniająca |
| Responsywność | Wysoki | Dostępność na każdym urządzeniu |
| Filtry i wyszukiwarka | Wysoki | Użyteczność przy przeglądaniu |

### SHOULD HAVE (zrealizowane)
| Funkcja | Priorytet biznesowy | Uzasadnienie |
|---|---|---|
| Stats Section | Średni | Dodaje credibility i kontekst |
| Discover (ciekawostki) | Średni | Zwiększa engagement |
| Keyboard navigation | Średni | Accessibility wymóg |
| Section Memory | Niski | Nice-to-have UX |
| Forest Particles | Niski | Wow factor, ale nie krytyczny |

### WON'T HAVE (na razie)
| Funkcja | Powód |
|---|---|
| Dark Mode | Dodatkowy effort, niska wartość dla grupy docelowej |
| Quiz Leśny | Wymaga logiki quizowej, osobny projekt |
| PWA/Offline | Overkill dla strony edukacyjnej |
| Share button | Brak social sharing w tej grupie docelowej |
| Wiele języków | Grupa docelowa: polscy uczniowie |
